Common Issues

Issue 1: No Console Logs

Problem: No logs appear when saving mood

Solution: Check if user is logged in

Issue 2: Empty Arrays

Problem: Emotions/Activities arrays are empty: []

Solution: Check UI selection logic

Issue 3: No Hashtags in Journal

Problem: Journal entry doesn't contain hashtags

Solution: Check MoodContext hashtag processing

Issue 4: Analytics Not Showing

Problem: Analytics show "No data available"

Solution: Check hashtag parsing in analytics components

Manual Database Check

Run this in Supabase SQL Editor to check saved data:

SELECT id, date, mood_score, journal_entry, created_at
FROM mood_entries 
WHERE journal_entry LIKE '%#%'
ORDER BY created_at DESC
LIMIT 5;

Expected Results

  • Console shows pattern arrays being processed
  • Journal entries contain hashtags like #Happy #Exercise
  • Analytics parse and display pattern data
