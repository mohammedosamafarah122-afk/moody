Test Emotional and Activity Patterns

Database Schema Check

Check if the mood_entries table has emotions and activities columns:

-- Run this in Supabase SQL Editor:
SELECT column_name, data_type, is_nullable 
FROM information_schema.columns 
WHERE table_name = 'mood_entries' 
AND column_name IN ('emotions', 'activities');

Test Data Insert

Test inserting data with emotions and activities:

-- Run this in Supabase SQL Editor (replace with your user_id):
INSERT INTO mood_entries (user_id, date, mood_score, emotions, activities, journal_entry)
VALUES (
    'your-user-id-here',
    CURRENT_DATE,
    4,
    ARRAY['Happy', 'Excited'],
    ARRAY['Exercise', 'Music'],
    'Test entry with patterns'
);

Test Data Retrieval

Check if data is retrieved correctly:

-- Run this in Supabase SQL Editor:
SELECT id, date, mood_score, emotions, activities, journal_entry
FROM mood_entries 
WHERE emotions IS NOT NULL OR activities IS NOT NULL
ORDER BY created_at DESC
LIMIT 5;

Console Debugging

Open browser console and look for these logs when testing:

  • "Starting addMoodEntry..." - Function called
  • "Emotions array: [...]" - Emotions being saved
  • "Activities array: [...]" - Activities being saved
  • "Save successful: {...}" - Database save result
  • "Fetched mood entries: [...]" - Data retrieval

Common Issues

  • Database not updated: Run the SQL schema in Supabase
  • Authentication issues: Check if user is logged in
  • Array format: Ensure emotions/activities are arrays, not strings
  • RLS policies: Check Row Level Security policies
